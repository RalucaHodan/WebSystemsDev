    Abstract

    Mad Trivia is a trivia quiz application developed in JavaScript, using Angular Framework and JSON services that connect to Open Trivia Database API in order to get all the questions from the library and offer them to the player. The user can select the game mode they want to play by choosing from the quiz options, such as category, difficulty and the number of questions. The user is entertained by the variety of questions, the trivia application will let them answer the question in a 10 second time limit and share their final score with other users.

    Structure

    Overview

    Mad Trivia lets the user to choose the type of trivia game they want to play. From picking the questions from over 20 categories such as Entertaiment, Science, Sports, Arts, History, Geography, Math etc. or even the possibility of choosing all categories at once, difficulity levels and the number of questions they would like to answer. It has search options and possibility to listen offline. This application lets the user share their score and invite their friends from Facebook.

    Web app design and interaction

    Mad Trivia has plenty of options for the trivia enthusiast. The user can find the path to no more boredom from this game application. The page is structured into 4 categories: The Home page, The Game Page(Mad Trivia), where the user can generate the game, The Rules Page, where all the rules of the game are presented and The Profile Page, where the user can see their scores, leaderboard, time elapsed and history of games played.

    The Home Page

  • Welcome user and sign in/register

    The user is displayed a welcome message “Welcome! Lets play Mad Trivia!” and has two options: to log in or to register an account in case the user is a new gamers to our page. For the registration (sign up) there are 2 opportunities to sign up with their social media account,Facebook, or by Google account. By having an account, the user can save their preffered categories of questions, score and history of played games & answered questions.

    The registration requires: email, username, password, confirm password and terms and condition checkbox and a link to them.

  • The game

    After registration, a dialog shows up with the word "Play". The user will click on it and it will take them to another dialog where the user can Select Quiz Options and click Generate Game.

  • Mad Trivia

    The Select Quiz Options offer the user a personalised experience. They have to choose one of the Categories such as Arts, Sports, General Knowledge or they can select "Any Category" which sums up all the categories together. Difficulty levels come in 3: easy, medium and hard. Then the user can select the amount of questions they would like. The number of questions is limited to 20 due to the database. Then the user can click "Generate Game".

  • The gameplay

    Now the game begins. The questions are true or false questions with 2 radio buttons to pick the answer. Only one of the answer is correct. There will be a time elapsed that will show in the top left corner of the game. Once the user clicks on an answer and if its correct a pop up will show that the answer is correct and it will highlight in green. A score will be presented, of how many right questions and the end of the game after ther user clicks submit. An easy level question is worth 1000 points, a medium level question is worth 2500 points and a hard level question is worth 4000 points. If the user clicks on a wrong answer, the points for that question will not be added and the user will see at the end of the game how many questions they got wrong. The game is over once all the questions are answered and the user presses "Submit".Total score and total time played are shown.

  • Share your score

    At the end of the game, the user can share their score via Facebook and also see where they position themselves.

    • Methods

    In order to develop this application for the users, the technical structure must be set. This application is based on JSON services and API and the data stored in XML format from a database. The website client is developed in JavaScript using Angular Framework.

    The Open Trivia Database API is JSON-based and uses XML-formatted responses.

    The Open Trivia Database trivia questions resource returns information about the categories, questions of categories, difficulty levels and the number of questions selected.

    Open Trivia Database is a social website which has users from all over the world submitting Trivia questions. It keeps track of its users’ categories, and offers over 4500 verified questions from various categories to its players. Questions are being submitted daily and there are over 7000 questions pending approval to be introduced into the database. Most importantly, Opent Trivia database has an impressive API which developers can use to build games on web or mobile. It allows the gathering of information on users, categories, and questions.
